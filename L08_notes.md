## 一、假设检验回顾 (Slides 3–5)

> **讲师补充**: 本讲先把期中考前剩下的假设检验内容讲完，再进入估计部分。

### 假设检验基本步骤 (Slide 3)

1. **从零假设 $H_0$ 出发**，确定在零假设下检验统计量的分布
2. **计算 p 值**: 将观测数据与零假设下的期望进行比较
3. **判断**: 如果 p 值 ≤ 预设的显著性水平 $\alpha$，则拒绝 $H_0$
   - 常用 $\alpha = 0.05$，有时用 $0.01$

> **讲师解释**: p 值就是在零假设下，观察到与实际数据一样极端或更极端的概率。如果 p 值很小，说明数据在零假设下是"不寻常的"，构成对 $H_0$ 的证据。等价地，我们也可以设定一个拒绝域 (rejection region)，当统计量落入该区域时即拒绝 $H_0$。

### 两类错误与检验质量 (Slide 4)

|  | $H_0$ 为真 | $H_0$ 为假 |
|---|---|---|
| 不拒绝 $H_0$ | 正确 (True Negative) | **第二类错误** (False Negative) |
| 拒绝 $H_0$ | **第一类错误** (False Positive) | 正确 (True Positive) |

- 如果 $H_0$ 为真，我们希望 $\mathbb{P}(\text{reject})$ 尽量**小**
- 如果 $H_0$ 为假，我们希望 $\mathbb{P}(\text{reject})$ 尽量**大**（即功效/power 要高）

### Type I/II 错误的权衡 (Slide 5)

- 在零假设下拒绝的概率称为检验的**水平/大小 (level/size)**，记为 $\alpha$：

$$\mathbb{P}_{H_0}(\text{reject}) = \alpha$$

- 我们**先控制 Type I 错误为 $\alpha$**，然后在此约束下尽量**最大化功效 (power)**
- 通常 $\alpha = 0.05$

> **讲师强调**: 我们的测试框架（基于 p 值或拒绝域）在统计量为连续分布时确实能实现精确的 Type I 错误控制。离散分布情况下会有"离散化问题"，但只要分布足够细，近似效果也很好。

## 二、硬币抛掷例子（详解）(Slides 6–10)

### 问题设定 (Slide 6)

- 观察 200 次独立抛硬币，$X_i \sim \text{Bernoulli}(p)$
- $H_0: p = 1/2$（硬币是公平的）
- 检验统计量: $T = \text{正面次数} \sim \text{Binomial}(200, p)$
- 若 $H_1: p > 1/2$，则"极端"意味着 $T$ 很大
- 理想情况: 找临界值 $t$ 使得 $F_T(t) = 1 - \alpha$，当 $T > t$ 时拒绝

**R 代码**:
```r
qbinom(0.95, size=200, prob=0.5)  # [1] 112
pbinom(112, size=200, prob=0.5)   # [1] 0.9615812  ← 不完全是 0.95
```

> **讲师解释**: 因为二项分布是离散的，CDF 是阶梯函数，可能找不到恰好使 CDF=0.95 的值。`qbinom(0.95,200,0.5)` 返回 112，但 $P(T \le 112) = 0.9616$，并非恰好 0.95。

### 调整临界值 (Slide 7)

取 $t = 111$：$P(T \le 111) = 0.9482 \approx 0.95$，更接近目标。

**拒绝规则**: $T > 111$ 时拒绝 $H_0$

**MC 模拟函数**:
```r
run_coinflip_trial <- function(pheads=0.5) {
  number_of_heads <- rbinom(n=1, size=200, prob=pheads)
  if (number_of_heads <= 111) {
    return(0)  # 不拒绝
  } else {
    return(1)  # 拒绝
  }
}
```

### MC 验证 Type I 错误 (Slide 8)

```r
NMC <- 1e4
nreject <- 0
for (i in 1:NMC) {
  nreject <- nreject + run_coinflip_trial()
}
nreject/NMC  # [1] 0.0534
```

结果约 5.34%，接近预设的 $\alpha = 0.05$，说明 **Type I 错误得到了控制**。

> **讲师说明**: "Nice" 的含义是：在零假设下，拒绝概率约为 5%，符合我们的设计目标。

### 检验 Type II 错误 — $p = 0.75$ (Slide 9)

当 $p = 0.75$（硬币偏向正面）时:
```r
# 将 pheads 改为 0.75
nreject/NMC  # [1] 1  ✓
```
拒绝比例为 1（100%），说明当 $H_0$ 为假且 $p$ 很大时，检验能几乎肯定地拒绝 $H_0$，**Type II 错误为 0**。

### 检验 $p = 0.25$ 的情况 (Slide 10)

当 $p = 0.25$（硬币偏向反面）时:
```r
# 将 pheads 改为 0.25
nreject/NMC  # [1] 0
```
拒绝比例为 0！

> **讲师重点**: 这**不代表我们"接受" $H_0$**。原因是我们选的是单侧检验 $H_1: p > 1/2$，拒绝域在右侧（$T$ 大时拒绝）。当 $p = 0.25$ 时 $T$ 会很小，根本不会落入右侧拒绝域。这是因为我们的拒绝规则是为了**在 $H_1$ 方向上优化 power** 而设计的。

## 三、单侧检验 vs 双侧检验 (Slides 11–17)

### 核心原理 (Slide 11)

- 给定 $\alpha$，把拒绝域放在**任意一侧**，概率为 $\alpha$，都能达到预设显著性水平
- 拒绝域的**方向**取决于 $H_1$ 的方向，因为我们想在 $H_1$ 下最大化 power

**双侧检验**: 若 $H_1: p \ne 1/2$：
- 拒绝条件: $T < t_{\alpha,1}$ 或 $T > t_{\alpha,2}$
- 需要: $\mathbb{P}(T < t_{\alpha,1} \text{ 或 } T > t_{\alpha,2}) = \alpha$

> **讲师解释**: 因为双侧备择假设说 p 可能大于也可能小于 1/2，所以我们需要在两边都设拒绝域，这样无论 p 偏大还是偏小都能检测到。

### 双侧临界值计算 (Slides 12–13)

对称分割 $\alpha$：每侧 $\alpha/2 = 0.025$

```r
qbinom(0.025, size=200, prob=0.5)  # [1] 86
pbinom(86, size=200, prob=0.5)     # [1] 0.02798287
```
因此 $t_{\alpha,1} = 87$，$t_{\alpha,2} = 114$

**双侧拒绝规则**: $T < 87$ 或 $T > 114$ 时拒绝

### 双侧 MC 模拟 (Slides 14–15)

```r
run_coinflip_trial2 <- function(pheads=0.5) {
  number_of_heads <- rbinom(n=1, size=200, prob=pheads)
  if (number_of_heads < 87 | number_of_heads > 114) {
    return(1)  # 拒绝
  } else {
    return(0)  # 不拒绝
  }
}
```

MC 结果: `nreject/NMC = 0.0492`，也约 5%，Type I 错误同样受控。

### 两种拒绝规则的比较 (Slides 16–17)

定义 `compare_reject_rates(p)` 函数，对不同 $p$ 值运行 $10^4$ 次 MC 实验，分别记录单侧和双侧的拒绝率。

**结果**（功效曲线图）:
- **单侧检验**（红线）: 当 $p > 0.5$ 时拒绝率迅速上升到 1；当 $p < 0.5$ 时拒绝率接近 0
- **双侧检验**（蓝线）: 当 $p$ 向**任一方向**偏离 0.5 时，拒绝率都迅速上升到 1

**结论**:
- 单侧检验在 $p$ 大时更易正确拒绝（单侧 power 更高）
- 双侧检验在 $p$ 小时更易正确拒绝
- 但在 $p > 0.5$ 的方向上，单侧检验的 power 略优于双侧

> **讲师总结**: 选单侧还是双侧取决于你的 $H_1$ 方向。这就是"by design"的结果。

## 四、检验统计量的选择 (Slides 18–20)

### 不同的"极端" (Slides 18–20)

以 $T = \text{正面次数}$ 为检验统计量，能检测"正面太多/太少"的极端情况。

但考虑另一种极端: **100 次连续正面 + 100 次连续反面**

- 此时 $T = 100$（正面数完全正常），但数据显然不像来自公平硬币
- $T$ **无法捕捉这种极端**

> **讲师指出**: 检验统计量是高维数据的一维摘要，不可避免会丢失信息。没有唯一"最佳"的检验统计量，选择取决于你关心的极端类型。

可以用其他统计量，如**最长连续正面长度** (`longestRun` 函数):

```r
longestRun = function(x, target){
  max(0, with(rle(x), lengths[values==target]))
}
```

若最长连续正面 ≥ 100，也可认为数据极端。

## 五、选择 $\alpha$ 的权衡 (Slides 21–22)

- $\alpha = 0.05$ 是历史惯例，有时也用 $0.01$
- **没有唯一"正确"的 $\alpha$**，取决于应用场景中**假阳性的代价**

### 癌症筛查例子 (Slide 21)
- $H_0$: 患者没有癌症；$H_1$: 患者有癌症
- **Type I 错误**: 错误地说患者有癌症 → 患者焦虑，需做额外检查
- **Type II 错误**: 漏检癌症 → 后果非常严重

> **讲师分析**: 在这个场景下 Type II 错误更严重。因此应适当提高 $\alpha$（容忍更多 Type I 错误），以降低 Type II 错误。

### 一般规律 (Slide 22)
- **$\alpha$ 越低** → Type I 错误越少，但 Type II 错误越多
- **$\alpha$ 越高** → Type I 错误越多，但 Type II 错误越少

## 六、估计 (Estimation) 入门 (Slides 23–24)

> **讲师过渡**: 到此假设检验部分结束。接下来进入一个全新的章节——估计 (Estimation)。估计和假设检验一样，是统计学最基本的任务之一。

**估计 (Estimation)** 的定义: 利用数据给出一个值或一个值的范围，作为对某个未知量的"好的猜测"

可以估计的对象:
- 模型的参数（如分布的均值）
- 事件发生的概率（可用 Monte Carlo 方法）

---

## 七、例子：估计人类平均身高 (Slides 25–29)

- **总体**: 地球上所有人
- **目标量**: 平均身高 $\mu$

### 估计过程

1. 测量所有人的身高不现实
2. 取样: 抽取几千人，测量身高 $X_1, \ldots, X_n$
3. 用样本均值估计:

$$\hat{\mu} = \bar{X} = \frac{1}{n}\sum_{i=1}^{n} X_i$$

4. 例如 $\hat{\mu} = 172.1$ cm → "我们估计平均身高为 172.1 cm"

### 为什么 $\hat{\mu}$ 合理？ (Slides 27–29)

1. **可行性**: $\hat{\mu}$ 是观测数据的函数，可以计算
2. **大数定律 (LLN)**: 当 $n$ 足够大时，$\hat{\mu}$ 趋近于真实总体均值 $\mu$

> **讲师说明**: 虽然 n 只有几千（远小于全球人口），但只要身高分布足够规则，样本均值就是对总体均值的好近似。

## 八、点估计的正式定义 (Slides 30–34)

### 核心概念

**点估计 (Point Estimation)**: 用样本数据计算**一个值**来估计/猜测未知的总体参数 $\theta(P)$

- $P$ = 未知的总体分布
- $\theta(P)$ = 我们感兴趣的参数（如均值 $\mu$、方差 $\sigma^2$ 等）

### 估计量 (Estimator) vs 估计值 (Estimate)

| 概念 | 定义 | 例子 |
|---|---|---|
| **估计量 (Estimator)** | 样本（随机变量）的函数 $\hat{\theta}(X_1, \ldots, X_n)$ | $\hat{\mu} = \frac{1}{n}\sum X_i$ |
| **估计值 (Estimate)** | 将实际观测值代入估计量得到的具体数值 $\hat{\theta}(x_1, \ldots, x_n)$ | $\hat{\mu} = 172.1$ cm |

> **讲师类比**: 估计量和检验统计量的区别仅在于用途。用于假设检验的叫"检验统计量"，用于估计的叫"估计量"。它们都是"统计量 (statistic)"——数据的函数。

### 关键约束

**估计量不能包含任何未知参数！** 比如要估计 $\mu$，$\mu$ 本身不是一个合法的估计量，因为我们无法从数据中计算 $\mu$。

> **讲师强调**: 估计量必须是你能从手头数据算出来的东西。

## 九、点估计的完整流程图 (Slides 35–42)

### 估计均值 $\mu$ 的流程 (Slides 35–39)

```
数据生成过程 (未知总体 P)
    ↓ θ(P) = μ = E[X] ← 目标参数
    ↓ 生成数据
样本 X₁, X₂, ..., Xₙ (随机变量, 各自有分布)
    ↓ 实现为 x₁, x₂, ..., xₙ (具体数值)
    ↓ 用函数汇总
估计量: μ̂ = (1/n)∑Xᵢ (随机变量, 有自己的抽样分布)
    ↓ 代入实际值
估计值: (1/n)∑xᵢ = 172.1 cm (一个具体数字)
```

> **讲师解释**: 估计量本身也是随机变量（因为 $X_i$ 是随机的），所以它有自己的**抽样分布 (sampling distribution)**。根据 LLN 和 CLT，当 n 充分大时，$\hat{\mu}$ 的抽样分布会集中在真值 $\mu$ 附近。

### 估计方差 $\sigma^2$ 的流程 (Slides 40–42)

目标: $\theta(P) = \sigma^2 = \text{Var}(X)$

估计量: **样本方差**

$$s^2 = \frac{1}{n-1}\sum_{i=1}^{n}(X_i - \bar{X})^2$$

- 注意分母是 $n-1$（无偏校正）
- $s^2$ 本身也是随机变量，有抽样分布（与卡方分布相关）
- 代入实际数据 $x_1, \ldots, x_n$ 得到估计值

## 十、课堂练习 (Slides 43–44)

**问题**: $X_1, \ldots, X_n$ iid 来自未知分布 $P$（均值为 $\mu$），以下哪些是 $\mu$ 的**合法估计量**？

1. $X_1$ → ✅ 合法（是数据的函数，且 $E[X_1] = \mu$，无偏）
2. $(X_1 + X_2)/2$ → ✅ 合法（数据的函数，$E[(X_1+X_2)/2] = \mu$，无偏）
3. $\bar{X} = \frac{1}{n}\sum X_i$ → ✅ 合法（数据的函数，最常用的估计量）
4. $\mu$ → ❌ **不合法**（$\mu$ 是未知参数，无法从数据计算）

> **讲师解释**: "合法"意味着你能从数据中计算出来。$X_1$ 虽然只用了一个样本点，但它确实是数据的函数，且期望等于 $\mu$，所以是合法的（只是不如 $\bar{X}$ 好）。$\mu$ 本身不是数据的函数，所以不是合法估计量。

## 十一、Universal Widgets of Madison 例子 (Slides 45–59)

### 问题描述 (Slides 45–47)

- UW-Madison 公司生产 widgets
- 每个 widget 独立地以概率 $p$ 正常工作，$1-p$ 不工作
- **目标**: 从生产线抽样，估计 $p$

### Step 1: 建立模型 (Slides 48–49)

- 样本: $X_1, \ldots, X_n$，其中 $X_i \sim \text{Bernoulli}(p)$（iid）
- 目标: 从数据 $X_1, \ldots, X_n$ 估计 $p$

### Step 2: 构造估计量 (Slides 50–52)

因为 $E[X_i] = p$（Bernoulli 分布的期望就是成功概率），所以:

$$\hat{p} = \frac{1}{n}\sum_{i=1}^{n} X_i = \text{正常工作的比例}$$

根据 **LLN**: $\hat{p} \xrightarrow{n \to \infty} p$

> **讲师说明**: 成功概率 = 期望值，这是 Bernoulli 分布的好性质。所以用样本平均（即成功比例）来估计 p 是自然且合理的。

### 模拟验证 (Slides 53–59)

**模拟代码**:
```r
n <- 200; p <- 0.8
generate_widgets <- function(n, p) {
  return(rbinom(1, size=n, p))
}

NMC <- 1000
functional_widgets <- rep(NA, NMC)
for (i in 1:NMC) {
  functional_widgets[i] <- generate_widgets(n, p)
}
hist(functional_widgets/n)
abline(v=p, col='red', lwd=4)
```

**结果**: 直方图集中在真值 $p = 0.8$ 附近，说明 $\hat{p}$ 是一个好的估计量。

> **讲师总结**: 一个"好"的估计量意味着在反复的随机实验中，估计值集中在真实参数附近（直方图围绕 $p$ 集中）。
